## Implementation Notes

本变更采用证据驱动删除：只有在静态引用扫描、框架入口检查和测试验证都支持“无用”的情况下才删除类。

判定一个类可删除时需要同时满足：

- Java 源码中不存在正常类型引用、构造调用、静态成员引用或继承实现关系。
- 不属于 Spring 组件扫描入口、配置类、过滤器、拦截器、AOP、MyBatis mapper/type handler、序列化 DTO、异常处理、枚举常量契约或第三方框架通过类名反射加载的目标。
- 删除后 Maven 编译与相关测试通过。

执行顺序：

1. 对后端生产类建立候选清单，优先使用 IDE/编译器证据、`rg` 引用扫描和 Maven 编译结果交叉验证。
2. 对每个候选类检查注解、配置文件、XML、字符串类名和框架约定入口，排除存在反射或运行时入口的类。
3. 删除确认无用的类，修复残留导入或包结构影响。
4. 运行后端测试；若测试失败，先定位根因，只保留可验证的安全删除。

不做事项：

- 不重命名仍在使用的类。
- 不改变控制器、服务、Mapper、DTO 或配置的 public contract。
- 不调整数据库 schema 或初始化数据。
- 不引入新的依赖或架构抽象。
